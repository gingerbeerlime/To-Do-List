**코드 리뷰 수정사항
1. 함수명은 길더라도 주석없이 알 수 있도록 짓는 것이 좋음
2. 가능한 변수, 이벤트, 메소드는 분리 시켜서 작성 
3. children[idx]같은 숫자값 들어간 부분 수정 -> html에 요소가 변경될시 유지보수가 어려움 
4. 반복해서 쓰이는 css값은 상수로 빼서 사용 
5. js 파일 분리 (ui컴포넌트별 or 기능별)
6. 주석은 꼭 필요한 곳만, 주석 없이도 코드가 이해가 되도록 작성 
7. webpack 설정 - babel 라이브러리 / webpack dev server / webpack cli > init > 선택값 
8. readme 작성 : 프로젝트 설명(기능 구현 리스트, 애플리케이션 설명, 설계)
